Data Model

Describes three things about the stored data:

  1. Structure
  2. Operations
  3. Constraints
  • Data is stored in two dimensional tables called relations.
  • Each row is a tuple representing a single instance of the entity.
  • Columns are called attributes
  • Every component in a tuple must have value taken from the attributes domain (specified data type)
Candidate Keys

A set of attributes for which each tuple in the relation must have a unique set values, and for which no subset of the set has this property. This property must hold for all possible instances of the relation.

Primary Keys

A chose of the candidate keys. There may be many candidate keys, but only one chosen primary key.

Foreign Keys

Two tables (relations) are linked using a shared key that is the primary key in one of the relations. The primary key of the other table that is shared is known as this tables foreign key. Every foreign key must be the value of the corresponding primary key in some tuple. Thus, the foreign key associates each tuple with exactly one tuple in the relation that is references.

Referential Integrity

Every foreign key value must appear as the value of the primary key in some row of the table it references. This restricts the changes that can be made:

  • We can add a row containing a foreign key only if the value of the foreign key appears among the values of the referenced primary key
  • We can remove a row containing a primary key only if the value of the primary key does not appear among the values of any referencing foreign key
Constraints

Maintained by the DBMS:

  • Domain: data type of attributes
  • Key constraints: candidate keys must be unique
  • Entity Integrity: each set of tuples must have a unique set of values in its primary key and no NULLs
  • Referential Integrity: Every foreign key must appear as the value of the primary key in some tuple of the relation (table) it references.
Instance V. Schema

Schema

  • Lists the attributes and domain of each relation
  • Arrows drawn each foreign and primary key it references. Instance
  • A satisfying implementation of the schema.
