source: https://www.youtube.com/watch?v=UQ-nXHwdSFU Google Remote Procedure Call created to facilitate communication between distributed systems so clients can call methods store on a server as if they were local. Built with Protobuf a language and platform neutral way to serialize data that can be translated to any languages function signatures. Under the hood gRPC relies on HTTP/2 for:

  • multiplexing - a single TCP connection can handle many RPC calls (concurrently).
  • header compression
  • duplex streaming Note, REST APIs are built on HTTP 1.1
